# warmup 目录文件说明（避免同名混淆）

## 数据：只认这一份

| 正确位置 | 错误位置（勿用） |
|----------|------------------|
| `warmup/data/cora.pkl` | ~~`warmup/cora.pkl`~~（若解压到根目录，请**移动**到 `data/`） |

`gcn.py` / `explore_cora.py` 均读取 **`data/cora.pkl`**（相对本目录）。

## 易混淆文件

| 文件 | 用途 |
|------|------|
| `README.md` | 头歌下发的赛题说明（可覆盖，勿删） |
| `data/README.md` | 本地补充：如何在没有官方包时用脚本生成 pickle |
| `gcn_official_template.py` | 头歌官方空模板（只读备份） |
| `gcn_custom_v1.py` | 早期重写版（不用于提交） |
| `gcn.py` | **正式版**：在官方模板上填完 TODO |
| `explore_cora.py` | 阶段 A 探查，不提交 |
| `scripts/build_cora_pkl.py` | 仅本地从 Planetoid 造数据，**不要**覆盖官方 `data/cora.pkl` |
| `result.json` / `result.zip` | 训练输出；换官方数据后需**重新运行** `gcn.py` 再打包 |

## 提交 zip 内容

仅包含根目录下的：

- `gcn.py`
- `result.json`

**不要**把 `cora.pkl` 打进 `result.zip`。

## 官方数据特点（已探查）

- train/val/test 共 **1640** 个有 mask 的节点（另有 1068 个节点无 mask，属正常，非错误）
- 测试集 `y` 全为 **-1**
